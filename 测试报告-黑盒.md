## 3. 测试方法

### 3.1 等价类划分 (Equivalence Class Partitioning)

将输入域划分为有效等价类和无效等价类，每类选取代表性值。

| EC10: 扩展拉丁 | 有效 | `üser@example.com` | True |

## 5. 发现的关键行为特征

### 5.1 @validator 装饰器的异常捕获

`@validator` 装饰器会捕获 `ValueError`、`TypeError`、`UnicodeError`，并返回 `ValidationError` 而非抛出异常：

- `between(5, min_val=10, max_val=1)` → 返回 `ValidationError`（不抛 ValueError）
- `between(5, min_val="a", max_val="z")` → 返回 `ValidationError`（不抛 TypeError）
- `length("test", min_val=-1)` → 返回 `ValidationError`（不抛 ValueError）

### 5.2 Cron 表达式的格式限制

当前实现不支持以下组合格式：
- **范围+步长**: `0-30/5` → ValidationError（标准cron应支持）
- **列表+范围混合**: `1,5,10-20` → ValidationError（因为解析顺序：先检查`-`再检查`,`）

### 5.3 AbsMin/AbsMax 比较行为

`AbsMin() <= AbsMin()` 返回 `True`（而非 False），因为 `__le__` 检查的是 `other is not AbsMin`（类），而实例不是类本身。

### 5.4 MAC地址严格的分隔符规则

不允许混合使用 `:` 和 `-` 分隔符，这是通过显式检查实现的（PR #432 修复）。

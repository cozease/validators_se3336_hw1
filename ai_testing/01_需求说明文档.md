## 1. 项目概述

`python-validators` 是一个无需 schema 定义、面向通用数据格式的 Python 数据校验库。
被测版本 `0.35.0`，源码 3 143 行（含 docstring），覆盖 7 大类、20+ 条校验函数。
所有校验函数通过统一装饰器 `@validator` 封装：

- 校验通过 → 返回 `Literal[True]`
- 校验失败 → 返回 `ValidationError`（可布尔化为 `False`；调用方可用 `bool()` 或 `is True` 区分）
- 显式开启 `r_ve=True` 或环境变量 `RAISE_VALIDATION_ERROR=True` 时，失败改为抛出异常

## 2. 功能需求（按模块）

### FR-01 邮箱验证 `validators.email`

**签名**
```python
email(value, /, *, ipv6_address=False, ipv4_address=False,
      simple_host=False, rfc_1034=False, rfc_2782=False)
```

**业务规则**
- 必须恰好含 1 个 `@`。
- 本地部分长度 ≤ 64，域名部分长度 ≤ 253（RFC 1034/5321）。
- 支持扩展拉丁字符、dot-atom 与带引号字符串。
- `ipv4_address`/`ipv6_address` 为 `True` 时，域名需形如 `[ip]`。
- `rfc_1034=True` 时允许域名尾点。

**示例**
- 合法：`someone@example.com`，`user!#$%&'*+/=?^_\`{|}~-@example.com`
- 非法：`bogus@@`、`plainaddress`、`user@-lead.com`

---

### FR-02 信用卡号验证 `card_number / visa / mastercard / amex / unionpay / diners / jcb / discover / mir`

**业务规则**
- `card_number` 使用 Luhn 算法校验。
- 品牌函数在 Luhn 基础上再约束前缀 + 长度：
  - Visa：`^4`，长度 16
  - Mastercard：`^(51–55|22–27)`，长度 16
  - Amex：`^(34|37)`，长度 15
  - UnionPay：`^62`，长度 16
  - Diners：`^(30|36|38|39)`，长度 ∈ {14,16}
  - JCB：`^35`，长度 16
  - Discover：`^(60|64|65)`，长度 16
  - Mir：`^(220[0-4])`，长度 16

---

### FR-03 URL 验证 `validators.url`

**业务规则**
- 支持 scheme 白名单：`ftp/ftps/git/http/https/irc/rtmp/rtmps/rtsp/sftp/ssh/telnet`。
- netloc 必须通过 `hostname` 子校验；可含 IPv6/IPv4/端口/用户信息。
- path/query/fragment 受 RFC3986 字符集限制。
- URL 中不允许空白字符。
- `private=True/False` 控制是否接受私网 IP。

---

### FR-04 IP 地址验证 `ipv4 / ipv6`

**业务规则**
- `ipv4`：支持 CIDR、`strict` 模式要求形如 `a.b.c.d/p`；`private` 参数过滤私网/回环/广播段。
- `ipv6`：同上，支持 IPv4-mapped IPv6（如 `::ffff:192.0.2.128`）。
- 前缀越界（IPv4 > 32 / IPv6 > 128）应拒绝。

---

### FR-05 UUID 验证 `validators.uuid`

**业务规则**
- 支持 `str` 与 `uuid.UUID` 对象。
- 字符串需匹配 `^[0-9a-fA-F]{8}-([0-9a-fA-F]{4}-){3}[0-9a-fA-F]{12}$`。

---

### FR-06 MAC 地址验证 `validators.mac_address`

**业务规则**
- 允许 `:` 或 `-` 作为分隔符，六段十六进制。
- 大小写不敏感。

---

### FR-07 哈希值验证 `md5 / sha1 / sha224 / sha256 / sha384 / sha512`

**业务规则**
- 正则匹配对应长度（32/40/56/64/96/128）的十六进制字符串。
- 空串视为无效。

---

### FR-08 编码验证 `base16 / base32 / base58 / base64`

**业务规则**
- base16：`^[0-9A-Fa-f]+$`
- base32：`^[A-Z2-7]+=*$`（大写 + 数字 2-7，允许 `=` 填充）
- base58：比特币字母表，排除 `0OIl`
- base64：`(?:[A-Za-z0-9+/]{4})*(?:[A-Za-z0-9+/]{2}==|[A-Za-z0-9+/]{3}=)?`

---

### FR-09 数值区间 `validators.between`

**业务规则**
- 支持 `int / float / str / datetime`。
- 区间闭合 `[min_val, max_val]`；允许只给一端。
- `value=None` 返回 `False`。
- `min_val > max_val` 应报错。

---

### FR-10 字符串长度 `validators.length`

**业务规则**
- 基于 `between(len(value), ...)` 实现。
- `min_val / max_val < 0` 应抛 `ValueError`（由装饰器转为 `ValidationError`）。

---

### FR-11 域名 `validators.domain`

**业务规则**
- 兼容 IDN（punycode）与 Unicode 直写。
- `consider_tld=True` 时仅接受 IANA 官方 TLD。
- `rfc_1034`、`rfc_2782` 分别控制尾点与服务记录下划线。
- 不允许空白或连续下划线。

---

### FR-12 Slug `validators.slug`

**业务规则**：`^[-a-z0-9]+$`，不允许大写、下划线、空格与表情。

---

### FR-13 Cron 表达式 `validators.cron`

**业务规则**：五字段（分/时/日/月/周）；支持 `*`、区间 `a-b`、步长 `*/n`、列表 `a,b,c`。

---

### FR-14 IBAN `validators.iban`

**业务规则**
- 2 位国家码 + 2 位校验 + 11~30 位字母数字。
- 通过 mod-97 校验。

---

### FR-15 金融证券编号 `cusip / isin / sedol`

**业务规则**
- CUSIP：9 位，字符集 `0-9A-Za-z*@#`，加权校验。
- ISIN：12 位字母数字，前 2 位必须为字母。
- SEDOL：7 位字母数字；禁用元音 `AEIOU`；加权 `[1,3,1,7,3,9,1]` mod 10 = 0。

---

### FR-16 加密货币地址 `btc_address / eth_address / bsc_address / trx_address`

**业务规则**
- BTC：支持 P2PKH/P2SH（base58 + 双 SHA256 校验）、Segwit 正则估判。
- ETH / BSC：`0x` + 40 位 hex，或带 EIP-55 校验和。
- TRX：Base58Check，以 `T` 开头。

---

### FR-17 国家信息 `calling_code / country_code / currency`

**业务规则**
- `calling_code`：值须属于内置 `+X` 号段集合。
- `country_code`：支持 `alpha2 / alpha3 / numeric / auto`；`ignore_case` 控制大小写。
- `currency`：支持 ISO 4217 代码（默认跳过符号），`skip_symbols=False` 时允许 `€/¥/$` 等符号。

---

### FR-18 i18n 身份/税号

包含：`es_cif / es_nie / es_nif / es_doi / fi_business_id / fi_ssn /
fr_department / fr_ssn / ind_aadhar / ind_pan / ru_inn`。
详见对应源码 docstring。

---

### FR-19 核心装饰器 `validator` 与异常 `ValidationError`

**业务规则**
- 调用传入 `r_ve=True` → 失败抛 `ValidationError`。
- 环境变量 `RAISE_VALIDATION_ERROR=True` 起同样作用。
- 拦截 `ValueError / TypeError / UnicodeError`，统一包装为 `ValidationError`。
- `ValidationError.__bool__` = `False`，`repr == str`，`args` 字典记录入参。

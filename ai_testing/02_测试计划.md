# 《测试计划》— python-validators v0.35.0

> 本计划由 Cursor Composer Agent 依据《需求说明文档》生成，随后人工复核。
> Prompt：`ai_testing/prompts/02_gen_test_plan.md`。

## 1. 测试目标

- 验证 `python-validators` 0.35.0 的 20+ 条 validator 在功能、边界与异常输入下的正确性。
- 发现实现与文档/标准之间的偏差（潜在缺陷）。
- 提供可回放、可重复的自动化测试工作流，总体代码覆盖率 **≥ 90%**。

## 3. 测试策略

| 策略 | 说明 |
| --- | --- |
| 单元测试 | pytest + parametrize 对每个函数做组合化覆盖 |
| 等价类分析 | 输入域划分为"合法/非法/边界"三类，取代表元素 |
| 边界值 | 长度、数值上下界、字符集边界 |
| 异常路径 | 空串、None、类型错误、超长、非 ASCII |
| 正则对抗 | 构造正则前缀重叠/尾部冲突样本探测实现缺陷 |
| 覆盖率 | `pytest-cov` 统计行覆盖 |
| 回归对比 | 与既有 `tests/` 共同执行，防止 AI 测试破坏既有约束 |

## 6. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| LLM 生成非法代码 | 测试无法运行 | 生成后立即 `pytest --collect-only` 自检并迭代修复 |
| LLM 捏造需求 | 断言失真 | 断言仅来自源码 docstring；生成前强制先 Read 源码 |
| 覆盖率虚高 | 无法反映分支 | `--cov-report=term-missing` + 人工核对缺失分支 |
| 上游真实缺陷 | 误判为测试失败 | 使用 `xfail(strict=True, reason=…)` 透明暴露 |

## 7. 通过 / 失败准则

- **通过**：
  - 全量测试执行完毕，非 xfail 用例 0 失败。
  - 核心模块（email / url / ip / card / uuid / hashes / encoding / between / length）行覆盖率 = 100%。
  - 总体行覆盖率 ≥ 90%。
- **失败**：任一非 xfail 用例失败 或 总覆盖率 < 90%。

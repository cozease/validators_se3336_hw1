# Prompt — 生成《测试计划》

**Role**：测试经理。

**Input**：`ai_testing/01_需求说明文档.md`

**Task**：根据需求文档输出一份测试计划，涵盖：
1. 测试目标与范围（in / out of scope）
2. 测试策略：单元测试、等价类、边界值、参数组合
3. 测试环境：Python 版本、依赖、OS
4. 资源与进度表（迭代/里程碑）
5. 风险评估与应对
6. 通过/失败准则（退出条件，如覆盖率 ≥ 90%、阻塞级缺陷 = 0）
7. 交付物清单

**Constraints**：
- 每个功能点必须在"测试范围"小节可追溯到需求 FR-xx
- 使用 Markdown 表格展示进度
